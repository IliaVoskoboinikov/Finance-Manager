# Архитектура Авторизации и Управления Сессиями

Модуль `:core:auth` управляет жизненным циклом сессии пользователя, JWT-токенами и координирует доступ к сетевым ресурсам.

---

## 1. Состояния и Модели

### AuthStatus (Enum)
Базовое перечисление: `AUTHORIZED`, `GUEST`, `UNAUTHORIZED`.

### SessionState (Sealed Interface)
Атомарный снимок состояния системы. Используется для предотвращения Race Conditions:
*   `Authorized(access, refresh)` — гарантирует наличие токенов.
*   `Guest` — офлайн режим.
*   `Unauthorized` — сессия отсутствует.

---

## 2. Стейт-машина (`AuthStateHolder`)

Ядро системы (Single Source of Truth). Реализует **State-First подход**: состояние обновляется в памяти мгновенно, а запись на диск (DataStore) и в KeyStore происходит как побочный эффект.

### Ключевые принципы:
1.  **Suspend Events**: Метод `sendEvent` является приостанавливаемым. Это гарантирует, что вызывающий код (репозиторий) дождется завершения всех Side-эффектов (например, очистки БД) перед тем, как продолжить работу.
2.  **Атомарность**: Использование `Mutex` внутри машины гарантирует строгую последовательность обработки событий.
3.  **Синхронный доступ**: Благодаря хранению состояния в `MutableStateFlow` на диспетчере `Main.immediate`, сетевые интерцепторы получают актуальный токен мгновенно без `runBlocking`.

### Матрица переходов

| Событие (`AuthEvent`) | Исходное состояние | Итоговое состояние | Side Effects |
| :--- | :--- | :--- | :--- |
| **`OnLoginSuccess`** | `GUEST` / `UNAUTH` | `AUTHORIZED` | Очистка БД (при необходимости), сохранение токенов, обновление DataStore. |
| **`OnLogout`** | `AUTHORIZED` | `GUEST` / `UNAUTH` | Удаление токенов, очистка БД (опционально), сброс сессии. |
| **`OnSessionExpired`** | `AUTHORIZED` | `UNAUTHORIZED` | **Экстренная очистка:** удаление токенов и всех приватных данных БД. |
| **`OnClearData`** | *Любое* | *Без изменений* | Только вызов `DatabaseCleanupManager.clearUserData()`. |

---

## 3. Сетевой слой

### Система "Двойного Retrofit"
1.  **Main Retrofit**: Для бизнес-запросов. Включает `AuthInterceptor` (без `runBlocking`), `TokenAuthenticator` (с автоматическим рефрешем) и `GuestAccessInterceptor` (блокировщик).
2.  **Base Retrofit**: Только для `/auth/*`. Не имеет интерцепторов авторизации, что предотвращает рекурсивные дедлоки при обновлении токена.

### Безопасность (KeyStore)
Токены шифруются перед записью в хранилище с помощью `AES/CBC/PKCS7`. Ключи управляются системным модулем `AndroidKeyStore`, что обеспечивает защиту даже на Root-устройствах.

---

## 4. Навигация и UI
`RootNavGraph` реактивно подписан на `authStatus`. Любое изменение статуса на `UNAUTHORIZED` мгновенно сбрасывает стек экранов и переводит пользователя на форму входа. Это работает детерминировано как для ручного выхода, так и для фоновых ошибок синхронизации.
